# Alpha Sentinel OS - 용어 및 기술 가이드

Alpha Sentinel OS 개발 및 운영에 필요한 핵심 경제 용어와 기술(Docker) 개념을 정리한 문서입니다.

---

## 📊 1. 퀀트 & 거시경제 용어 (Economics & Quant)

이 프로젝트의 **Macro Sentry**와 **Sector Strategist**가 사용하는 핵심 지표들입니다.

### 1.1. 거시경제 지표 (Macro Indicators) - `fred.py`
| 용어 | 영문명 | 설명 | 투자 해석 |
| :--- | :--- | :--- | :--- |
| **기준금리** | **FEDFUNDS** | 미 연준(Fed)이 설정하는 초단기 금리. | 금리 인상 = 유동성 축소 (주식 악재)<br>금리 인하 = 유동성 공급 (주식 호재) |
| **10년물 국채** | **DGS10** | 미국 정부가 발행하는 10년 만기 채권 금리. | **무위험 수익률(Risk-Free Rate)**의 기준.<br>이것이 오르면 주식의 매력이 떨어짐 (자금 이동). |
| **장단기 금리차** | **T10Y2Y** | 10년물 금리 - 2년물 금리. | **가장 강력한 경기침체 경고등.**<br>정상 시장에선 장기 금리가 더 높아야 하는데, 이것이 **음수(-, 역전)**가 되면 6~18개월 뒤 불황이 온다는 신호. |
| **공포 지수** | **VIX** | CBOE Volatility Index. | S&P 500의 변동성 예측치.<br>- **20 미만**: 시장이 평온함 (Risk-On)<br>- **30 이상**: 시장이 공포에 질림 (Risk-Off, 투매 발생) |
| **실업률** | **UNRATE** | 경제 활동 인구 중 실업자 비율. | 실업률 급등 = 경기 침체(Recession) 시작.<br>너무 낮아도 '인플레이션' 우려로 금리를 올릴 수 있음. |

### 1.2. 투자 전략 용어 (Strategy)
| 용어 | 설명 | 우리 시스템 적용 |
| :--- | :--- | :--- |
| **Risk-On** | 시장 참여자들이 위험자산(주식, 코인)을 선호하는 상태. | VIX < 20, 실업률 안정 시 -> **적극 매수 (Sector Strategy 가동)** |
| **Risk-Off** | 시장이 공포에 질려 안전자산(달러, 채권, 금)으로 도망가는 상태. | VIX 급등, 금리차 역전 시 -> **현금 확보 (모든 매수 중단)** |
| **Momentum** | "오르는 놈이 계속 오른다"는 성질. 최근 수익률이 좋은 자산을 사는 전략. | 최근 1, 3, 6개월 수익률 가중 평균 점수로 **주도 섹터** 선정. |
| **Sector Rotation** | 경기 사이클에 따라 주도 업종이 바뀌는 현상. | 경기 회복기(Tech, Financial) <-> 경기 침체기(Healthcare, Staples) 자동 전환. |

## 🐳 2. Docker & 시스템 용어 (Tech Stack)

이 프로젝트는 컨테이너 기술인 Docker를 기반으로 구축되었습니다.

### 2.1. 기본 명령어 및 개념
| 명령어/용어 | 설명 | 상황별 사용법 |
| :--- | :--- | :--- |
| **Container** | 격리된 공간에서 실행되는 '앱' 하나. (Backend, DB 등) | VM(가상머신)보다 가볍고 빠름. 내 컴퓨터 환경을 타지 않음. |
| **Image** | 컨테이너를 찍어내는 '설계도'. (소스코드 + 라이브러리) | `requirements.txt`가 바뀌면 이미지를 다시 빌드해야 함. |
| **Volume** | 컨테이너가 삭제되어도 데이터가 남는 '저장소'. | DB 껐다 켜도 데이터가 안 날아가는 이유. (`postgres_data`) |
| **`docker-compose up -d`** | 여러 컨테이너를 백그라운드에서 한 번에 실행. | `-d`(Detached): 터미널을 차지하지 않고 뒤에서 조용히 실행. |
| **`--build`** | "설계도 다시 그려!" (이미지 재빌드). | 파이썬 라이브러리 추가하거나 `Dockerfile` 수정했을 때 필수. |
| **`docker exec -it`** | 실행 중인 컨테이너 안에 '접속'해서 명령 내리기. | 예: `docker exec -it alpha-sentinel-backend python ...`<br>(백엔드 컴퓨터 안에 들어가서 파이썬 실행해라) |

## 🤖 3. AI 에이전트 용어 (LangGraph)

| 용어 | 설명 | 역할 |
| :--- | :--- | :--- |
| **Agent State** | 에이전트들끼리 주고받는 '기억'이나 '메모장'. | Macro Agent가 "위험해!"라고 적어두면, CIO Agent가 그걸 보고 판단함. |
| **Node** | 특정 작업을 수행하는 전문가 에이전트. | `Macro Node`(경제 분석가), `Sector Node`(전략가). |
| **Edge** | 노드와 노드 사이의 연결선. (작업 순서) | Macro 분석 끝남 -> Sector 분석 시작. |
| **Pipeline (Chain)** | 여러 노드가 순서대로 연결된 전체 흐름. | 데이터 수집 -> 판단 -> 행동으로 이어지는 일련의 과정. |
